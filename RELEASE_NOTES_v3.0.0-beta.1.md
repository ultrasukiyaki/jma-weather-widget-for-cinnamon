# v3.0.0-beta.1

v3の耐障害性を固めるベータ第1版です。前回成功した天気データを永続キャッシュし、通信障害やAPI片側障害でもアプレットを空表示にせず、データの鮮度を明示して継続表示します。

## 追加

- `CacheService`によるlast-goodデータの永続保存
- 起動直後のキャッシュ即時表示
- 24時間を超えたキャッシュの自動破棄
- 地域設定シグネチャによる別地域キャッシュの混入防止
- Provider別の鮮度状態管理
- 「前回取得データ」「一部は前回取得データ」表示
- timeout／HTTP／JSON／通信／解析エラー分類
- 更新世代管理と通信中の再取得集約
- キャッシュ・部分障害・全障害の自動試験

## キャッシュ保存先

```text
~/.cache/jma-weather@10yendama.com/weather-<instance-id>.json
```

`XDG_CACHE_HOME`が設定されている場合は、その配下へ保存されます。

## 確認ポイント

- オンライン時に通常どおり更新され、キャッシュファイルが作成されること
- Cinnamon再読み込み直後、ネットワーク取得完了前に前回データが表示されること
- オフライン時に「前回取得データ（更新失敗）」と表示されること
- JMAまたはOpen-Meteo片側だけ失敗した場合、「一部は前回取得データ」と表示されること
- 通信中に「今すぐ更新」や設定保存を重ねても、古い地域・古い応答へ巻き戻らないこと
- キャッシュデータだけでは雨・高温・UV通知が発火しないこと
